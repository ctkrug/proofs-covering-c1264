Date: 2026-07-24. Frontier revision 4, manifest
`f4e5791954131995e29252e97a10884a0fccbc5e863e16bcc2d92a8f604aa4bc`
(`artifacts/portfolio/frontier-manifest-v1.json`), which entered this campaign at 33 closed / 14 open.

All 47 hash-pinned frontier nodes are now closed UNSAT. Under the forced exact-degree reduction below,
that excludes every 40-block cover, and with the known 41-block construction it pins the exact value:

```
C(12,6,4) = 41
```

## The reduction (why node UNSAT is decisive)

* A 40-block cover forces **every** point to degree exactly 20: each point lies in `C(11,3) = 165`
  quadruple-triples, which need at least `C(11,5,3) = 20` blocks, and total degree is
  `40 * 6 = 240 = 12 * 20`. No slack, so all degrees are exactly 20 — forced, not assumed.
* Pair multiplicities are likewise forced: `40 * C(6,2) = 600` over `C(12,2) = 66` pairs gives
  6 pairs at λ=10 (a perfect matching) and 60 pairs at λ=9.
* So the link of any point is a 20-block `C(11,5,3)` cover with exact degree vector
  `(10, 9,9,9,9,9,9,9,9,9,9)`, and the relabeling group is `C2 ≀ S5` of order 3840.
* The 47-node frontier partitions the exact-degree link search by root / secondary / tertiary orbit
  index (`scripts/run_cardinality_encoding_benchmark.py::non_cardinality_core`). Every 40-block cover
  therefore has a link inside some node. All 47 UNSAT ⇒ no 40-block cover.

## Soundness of the blocker

The link blocker removes known link orbits from the node search, which is what makes the hard nodes
tractable. Adding an orbit is sound **only** if that link provably cannot extend to a 40-block cover.
So each of the 20 orbits carries a *residual-extension* certificate: fix the link at point 1, ask SAT
whether the remaining 20 blocks can complete a 40-block cover with the forced pair degrees, and get
UNSAT with a replayed DRAT proof. All 20 are certified (`residual-certificates.json`).

Two consequences worth stating plainly:

* **Blocker strengthening is monotone.** The chain 9 ⊂ 13 ⊂ 14 ⊂ … ⊂ 20 was checked to be strictly
  nested clause-set inclusion, so a node closed under blocker-9 stays closed under blocker-20. The
  aggregate report re-checks this.
* **The construction track is subsumed, not skipped.** Every residual solve is literally "does a
  40-block cover with this link exist?" Not one of them — 20 orbit solves plus all 47 node solves —
  ever returned SAT. There is no 40-block cover to find.

## The nine base orbits were re-proved from scratch

The inherited base catalogue carried damaged provenance: `PROOF-INCIDENT.json`
(`"status": "invalidated_proof"`, `"checker_verdict": "s NOT VERIFIED"`), a `superseded_invalid_proof`
record, a `CERTIFICATE-INCIDENT.json` plus `ORCHESTRATION-INCIDENT.json`, and one `result.json` with
`status: UNKNOWN, proof: null`. Rather than repair that record, all nine residual-extension proofs
were regenerated with a fresh builder, a fresh CaDiCaL solve streaming DRAT, and an independent
re-reconstruction and re-replay (`scripts/reprove_base9.py`). 9/9 verified.

## Reproducing

```bash
python scripts/gen_cnfs_all.py --any-status            # rebuild node CNFs from blocker + manifest
cadical -t 10800 -q --no-binary <node>.cnf <node>.drat # expect exit 20 / s UNSATISFIABLE
drat-trim <node>.cnf <node>.drat -f                    # expect s VERIFIED
python scripts/audit_residual_indep.py <link> <cnf> <proof.gz>   # independent orbit checker
python scripts/aggregate_verify.py --replay            # whole-ledger report
```

Hardest node: `s-r0-2`, which needed seven rounds of orbit discovery (blocker 13 → 20) before it went
UNSAT — 643 s solve, 2.18 GB DRAT, 1,173 s replay.
